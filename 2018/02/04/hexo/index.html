建站记录

Posted on 2018-02-04 Edited on 2024-08-17 In hexo

时隔一年又搭了一次博客，上一次使用hugo是一次完全失败且糟糕的体验，不推荐使用海外的云服务器搭建博客，部署到github还是相对稳定靠谱的。搭这个博客的主要目的是为了让多年来学习总结的知识有一个归宿，当然也可以随时查阅。

Hexo 概述

在搭建博客初期有很多的想法，最终为了管理的方便选择用Hexo博客框架，它能够将Markdown格式的文档渲染成漂亮的网页，这样我们就可以在很短的时间内迅速的创建出网站的静态内容。

由于多次的更换操作系统，各操作系统配置本地环境如下：

  • macos
  • linux
  • windows

这样就下载了我们所有需要的本地环境，概括的讲，我们需要node作为本地js解析环境，需要npm作为包管理器，需要git将网站代码部署到github仓库当中。

构建与部署

我们可以利用GitHub提供的Pages服务来托管我们的博客。在GitHub Pages的首页上就有一个教程指导我们如何托管自己的网站，当然第一步得在GitHub上注册一个属于自己的账号，登录成功之后才能进行后续的操作。

  • 根据自己的用户名创建一个仓库，仓库一定要取名为用户名.github.io。
  • 修改博客项目的配置文件_config.yml，配置使用GitHub来部署该博客项目。

之后以一下命令部署:

Token

在2021年8月，Github在代码部署端弃用了帐号、密码的验证方式，转而使用token，因此在输入密码时需要输入tocken，github -- > settings --> Developer setting --> personal access tokens --> generate new token

可以通过更改密码缓存时间以避免重复输入，但是token仍有必要保存在本地。

域名

在初始版本中我使用了Godady + Cloudflare，目前阿里云提供了域名购买到解析的一套服务，对与无WWW域名，只需要在控制台配置A记录即可，写入github项目ip，并在本地项目source文件下添加CNAME文件保存域名，即可实现解析。

首先打开阿里云域名控制台，在域名列表中找到要使用的域名，点击解析，添加两个记录：
