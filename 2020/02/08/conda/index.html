管理虚拟环境

不同版本的库需要不同的编译器版本，通过conda创建不同的虚拟环境，每个环境都有不同的python版本，可以在激活环境后安装对应的库，从而形成多个封装好的并对应不同应用需求的环境。其关系如下图所示：

在conda中可以构建出多个相互隔离的环境，各个环境都可以由包管理器conda安装和卸载对应版本的包，方便的满足了我们的需求，比如Opencv在3.4.2版本后有些算法因被申请专利而无法使用，配置环境使用Opencv ver 3.4.1.15 + Python ver 3.6.3可以让我们专门构建出老版本。

Anaconda提供了更丰富的功能和插件，出于对命令行和开源的喜爱我使用Mini-conda，官方网站列出了各个操作系统对应的安装包。

conda换源

conda默认使用国外的安装源，国内网络存在安装时的time out问题，因此可以更换国内镜像，如阿里、清华镜像，其使用方式也会随conda版本发生变换，因此建议先查看官方网站，这里以清华源为例：

如上图所示应首先创建.condarc文件，运行conda config --set show_channel_url yes生成配置文件，在windows中在C:\用户\[自己的用户id]下，mac与linux可以直接输入cd ~&&ls -al找到.condarc文件，全部删除文件内容后将以下内容粘人即可。

使用

  • 创建一个新环境 conda create -n [name] python==[version]
  • 在conda activate [env]后，选取的环境被抽取出来作为当下的环境。
  • 安装所需的包 conda install [module]
